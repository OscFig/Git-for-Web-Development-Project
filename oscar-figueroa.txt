    37. Walk me through Lambda's git flow. 
    - Make a your own version of the repo.
    - Check if your name is in front of the repo before the slash
    - Clone your repo using the "Clone or download" button
    - In your CLI, use cd to go to the directory where you are going to want to clone your forked repo.
    - Use git clone <link-to-repo> command to clone the repo to the directory you are in.
    - cd into the newly created repo using the cd <repo-name> command
    - Use the <git checkout -b 'fifrstname-lastname'> command to create your own branch name for the repo.
    - Work on you repo and start getting the project assignment done.
    - Next, use <git add .>  to have the new changes ready for commiting
    - Type <git commit -m "Your commit message here"> this will commit the changes with your specified commit message
    - Push your changes using <git push -u origin branch-name> *make sure your branch name matches when you did <git checkout -b>
    - Now the changes are on github and you can submit a Pull Request
    - Go to your forked project repo on github * Check if you are on YOUR forked copy by clicking on the branch:master button,
        then click your name-branch
    - After that, go to the New pull request button to make a pull request on your name branch
    - It will redirect you to the pull request page
    - DO NOT CLICK THE Create pull request button until after you've finished the next step
    - when you are at the pull request page, click the dropdown button that says "base repository" and CHANGE IT to be YOUR forked repo
        instead of the original. *Look for GIT Username*
    - NOW, you can create the pull request.
